# 🚀 Big-Jay AI-OS

**The Ultimate Local & Cloud-Hybrid Agentic Operating System.**

Big-Jay AI-OS is a complete, containerized environment designed to seamlessly bridge local GPU inferencing (like RTX 4090) with bleeding-edge cloud models (GLM-5, Claude 4.6, Gemini 3.1 Pro). It provides a unified dark-mode Command Center to manage vector databases, workflow automations, and highly secure AI agent sandboxes.

---

## 🏗️ System Architecture

Big-Jay isn't just a chat app; it is a microservice ecosystem managed via a localized gateway.

* **Command Center GUI:** Custom Python (`customtkinter`) dashboard for single-click system ignition and sandbox routing.
* **AI Gateway (LiteLLM):** The "Traffic Cop" that translates and routes all API calls to local Ollama models or cloud providers, tracking token usage.
* **Main Interface (Open WebUI):** The primary chat and document retrieval frontend.
* **Visual Automations (n8n):** Drag-and-drop workflow builder for agent automation.
* **Vector Database (Qdrant):** Long-term memory and RAG storage for your documents.
* **Infrastructure Manager (Dockge):** A visual web GUI to manage, update, and deploy new Docker containers without touching the command line.
* **Secure Agent Sandboxes:** * **Windows Sandbox:** A localized Python virtual environment pre-loaded with CrewAI and spatial data tools (`geopandas`, `pyogrio`).
    * **Linux Sandbox (WSL):** NVIDIA NemoClaw integration for secure, isolated agentic code execution.

## 📋 Prerequisites

Before installing, ensure your host machine has the following:
* **Windows 10/11** with WSL2 enabled.
* **Docker Desktop** (Running and connected to WSL).
* **Python 3.10+** (Added to Windows PATH).
* *Optional but recommended:* A dedicated NVIDIA GPU (e.g., RTX 4090) and Ollama installed for local inferencing.

## ⚙️ Installation & Setup

**1. Clone the Repository**
```powershell
git clone [https://github.com/YOUR_USERNAME/Big-Jay-Ai-OS-Repo.git](https://github.com/YOUR_USERNAME/Big-Jay-Ai-OS-Repo.git) C:\Big-Jay-AI-OS
cd C:\Big-Jay-AI-OS
```

**2. Configure API Keys**
For security, API keys are never uploaded to this repository. You must create your local config file:
1. Rename `config.example.yaml` to `config.yaml`.
2. Open `config.yaml` and replace the placeholder text with your actual OpenRouter, Google AI Studio, or OpenAI keys. 

**3. Build the Python Environment**
Navigate to the Agents folder and install the core AI and spatial analysis dependencies:
```powershell
cd Agents
python -m venv venv
.\venv\Scripts\pip.exe install -r ..\requirements.txt
```

**4. Launch the System**
Double-click the `START-BIG-JAY.bat` file in the root directory. 
This will silently boot all Docker containers in the background and immediately launch the Big-Jay AI-OS Command Center GUI.

---

## 🖥️ Using the Command Center

Once the GUI is open, you have instant access to your entire ecosystem:
* **Power Control:** Boot or shutdown the background Docker engines.
* **Workspaces:** Jump straight into Open WebUI for chat, or n8n for automations.
* **Infrastructure:** Open Dockge to visually edit your `docker-compose.yml`, or Qdrant to inspect your vector memory.
* **Sandboxes:** Launch pre-activated PowerShell terminals for building CrewAI scripts, or drop straight into your Linux WSL NemoClaw container for secure agent runs.

---

## 🔒 Security Notes
* **API Keys:** Your `config.yaml` file is intentionally included in the `.gitignore` to prevent accidental uploads of your private API keys.
* **NemoClaw:** For untrusted code execution generated by AI models, always route the execution through the Linux (WSL) Sandbox to utilize containerized kernel security.
